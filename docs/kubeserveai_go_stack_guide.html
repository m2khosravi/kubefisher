The single most important Go rule for Kubernetes tooling

Use controller-runtime client.Client everywhere you talk to the Kubernetes API — not raw client-go, not dynamic client, not rest client. It has a built-in cache that reduces API server load dramatically, uses the same scheme as kubebuilder, and handles reconnection automatically. This is what every production operator uses in 2026.

Error handling ▶
01
Always wrap errors with context: return fmt.Errorf("patching annotation on %s: %w", name, err). Never return a bare err. When the error surfaces in logs, you need to know which operation failed.
02
Use client.IgnoreNotFound(err) when fetching resources in a reconcile loop. Resources can be deleted between the watch event and your fetch — this is expected, not an error.
03
Never panic in production code. Only panic in init() and main() where the program cannot meaningfully continue. Everywhere else: return the error and let the caller decide.
04
Return ctrl.Result{RequeueAfter: ...} not errors for expected retry scenarios in reconcilers. Returning an error triggers exponential backoff with jitter — good for unexpected failures. RequeueAfter is for scheduled re-checks like quota polling.
// ✓ correct — wrap with context if err := r.Status().Update(ctx, "a); err != nil { return ctrl.Result{}, fmt.Errorf("updating quota status: %w", err) } // ✗ wrong — bare error loses context if err := r.Status().Update(ctx, "a); err != nil { return ctrl.Result{}, err }
Logging — slog for binaries, logr for operators ▶
01
Cost patcher + CLI + agent: use log/slog with JSONHandler in production. It is stdlib, zero deps, and the 2026 ecosystem standard. Switch to TextHandler for local dev with LOG_FORMAT=text env var.
02
TeamInferenceQuota operator: use logr via log.FromContext(ctx). This is what controller-runtime expects. kubebuilder wires zap as the backend automatically.
03
Never use fmt.Println or log.Println in production code. Unstructured output cannot be filtered, indexed, or correlated with traces. Every log line must be a structured key-value pair.
04
Always include namespace and resource name in log fields: slog.Info("patched annotation", "namespace", ns, "resource", name, "cost_per_token", cost). These are your debugging fields in production.
// ✓ slog in cost patcher — structured, filterable logger := slog.New(slog.NewJSONHandler(os.Stdout, nil)) logger.Info("cost annotation written", "namespace", pod.Namespace, "resource", owner.GetName(), "adapter", adapter.Name(), "cost_per_token", fmt.Sprintf("$%.8f", costPerToken), ) // ✗ wrong — unstructured, useless in production fmt.Printf("wrote annotation to %s\n", name)
Kubernetes API patterns ▶
01
Always use server-side apply for writes: r.Patch(ctx, obj, client.Apply, client.ForceOwnership, client.FieldOwner("kubefisher")). Never use Update() — it will conflict with other controllers. Server-side apply is idempotent and handles conflicts correctly.
02
For annotation patches specifically use MergePatch: client.MergeFrom(original) → modify annotations → r.Patch(ctx, modified, patch). This is safer than full Update and only touches the fields you changed.
03
Status updates use a separate sub-resource: r.Status().Update(ctx, obj) not r.Update(ctx, obj). Mixing them causes conflicts. kubebuilder's +kubebuilder:subresource:status marker enforces this at the API level.
04
Reconcile loops must be idempotent. Running the same reconcile 10 times must produce the same result as running it once. Test this explicitly: apply resource, trigger reconcile 5 times, verify state is identical each time.
// ✓ server-side apply — the correct 2026 pattern func (r *Reconciler) patchAnnotations(ctx context.Context, obj client.Object, annotations map[string]string) error { patch := client.MergeFrom(obj.DeepCopyObject().(client.Object)) existing := obj.GetAnnotations() if existing == nil { existing = map[string]string{} } for k, v := range annotations { existing[k] = v } obj.SetAnnotations(existing) return r.Patch(ctx, obj, patch) }
Testing strategy ▶
01
Unit tests for pure logic: cost calculation, adapter detection, label parsing. Table-driven tests with testify/assert. These run in milliseconds and need no cluster.
02
Integration tests with envtest: sigs.k8s.io/controller-runtime/pkg/envtest runs a real Kubernetes API server binary in your test. kubebuilder scaffolds this. Test your reconcile loop against it — not against mocks.
03
Never mock the Kubernetes client. Mocks drift from real API behaviour and give false confidence. envtest is fast enough for CI (30–60s) and gives you real behaviour.
04
E2E tests with k3d: make cluster-up → apply resources → assert annotations appear. Run this in GitHub Actions on pull requests using the same Makefile you use locally.
// ✓ table-driven unit test func TestCostPerToken(t *testing.T) { tests := []struct{ name string costPerHour float64 tokensPerSec float64 want float64 }{ {"a10g busy", 1.24, 800, 1.24/800/3600}, {"zero tokens", 1.24, 0, 0}, // no divide by zero } for _, tc := range tests { t.Run(tc.name, func(t *testing.T) { got := computeCostPerToken(tc.costPerHour, tc.tokensPerSec) assert.InDelta(t, tc.want, got, 0.0000001) }) } }

Set up the toolchain before writing a single line of product code

5 hours spent on tooling saves 50 hours of debugging and bad habits. Every item below should be green in CI before your first real commit.

Linting — golangci-lint ▶
Installbrew install golangci-lint — do not use go install, version drift causes CI/local discrepancies
ConfigWrite .golangci.yml in repo root. Enable: errcheck, govet, staticcheck, exhaustive, noctx, sloglint. Disable: gochecknoglobals (too strict for CLI).
CIRun golangci-lint run ./... on every PR. Block merge if it fails. Zero exceptions.
sloglintSpecifically add sloglint — it catches common slog mistakes like unkeyed attributes and inconsistent key casing.
# .golangci.yml linters: enable: - errcheck # all errors must be handled - govet # go vet built-in checks - staticcheck # SA* checks - exhaustive # switch on enums must be exhaustive - noctx # http requests need context - sloglint # structured logging correctness - godot # comments end with period disable: - gochecknoglobals # cobra uses package-level vars issues: exclude-rules: - path: _test\.go linters: [errcheck] # test files allowed to ignore errors
